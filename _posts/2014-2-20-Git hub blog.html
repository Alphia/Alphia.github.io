本文分为两个主题： 1. why git blog 2. how to build git blog 搭建git blog 1. why git blog 有个名叫阮一峰的同学，写的一篇博客，给出了有说服力的理由：http://www.ruanyifeng.com/blog/2012/08/blogging_with_jekyll.html 2. how to build git blog 在http://pages.github.com/上有相当简明的步骤，搭出一个具有index.html的helloworld的页面。但这只是一小步，离一个随心所欲的博客，有很大的距离。 2.1 加一篇博客 2.1.1 在根目录下的目录_post，创建一篇新的博客：yyyy-mm-dd-blogname.html（命名方式不止这一种，但有限定，在jekyll官网上查找) 2.1.2 然后在index.html上加上新博客的地址，即可。 2.2 本地安装jekyll 上小节的方法，固然可行，但在push到github之前，基本属于不知道新博客长什么样子。模板固然可以在一定程度上解决这个问题，但如果想自定义一个模板，还是需要本地能预览这个页面。 2.2.1 安装gem $brew install gem 2.2.2 安装jekyll $ gem install jekyll 2.2.3 理论上这时候有可用的jekyll了，但仅仅在理想环境下，以上两条命令才能搞定 2.2.4 可能会出现的问题，以及解决方案 2.2.4.1 mac下gem需要先升级：$ gem update --system 2.2.4.2 mac下ruby版本可能是1.8.7，需先升级ruby至1.9.3。 2.2.4.1.1 安装rvm: $sudoash < <(curl -s https://raw.github.com/wayneeseguin/rvm/master/binscripts/rvm-installer ) 2.2.4.1.2 $ rvm install 1.9.3 --with-gcc=clang 2.2.4.1.3 $ rvm use 1.9.3 2.3 使用刚才搭建的博客的根目录下，$ jekyll build; jekyll serve。按照提示，即可查看 2.4 试着写了一篇新博客，发现界面很丑。这时候模板就很重要了，不想写可在网上找
